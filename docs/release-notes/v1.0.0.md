# Inspect Dependency Source v1.0.0 — draft release notes

> Draft only. Publishing, tagging, and creating a GitHub release require separate authorization.

## Why this release exists

Coding agents can reason confidently inside a project and still inspect the wrong
revision at a dependency boundary. This release gives them a conservative path to
the implementation the project actually uses: determine the resolved version,
reuse or acquire that exact source, verify its commit, and inspect it read-only.

## Highlights

1. **Exact refs fail closed.** A missing tag or commit never silently becomes the
   default branch.
2. **One user-level catalog serves every project and agent.** A matching source
   tree is downloaded once and then reused.
3. **The local dashboard keeps the catalog observable.** It shows provenance,
   verification, integrity warnings, storage and freshness metrics, and operation
   history without adding mutation endpoints.

## Install once, route from each project

Install the skill at user level:

```bash
npx skills add Tairitsua/inspect-dependency-source-skill --global
```

Then add this routing rule to a project-level `AGENTS.md` so an agent invokes the
skill when dependency implementation details become relevant:

```md
## Dependency source inspection

When analysis or debugging depends on third-party library or SDK implementation,
automatically use the user-level `inspect-dependency-source` skill, even if it was
not explicitly named. Resolve the exact version or ref used by this project, reuse
a matching catalog source before fetching, inspect the returned `source_path`
read-only, and never substitute the upstream default branch.
```

Start a new agent session after changing `AGENTS.md` so the new guidance is loaded.
Claude Code reads `CLAUDE.md`; projects using it can keep one source of guidance by
placing `@AGENTS.md` in a root `CLAUDE.md`.

Claude Code can also install it as a marketplace plugin:

```bash
claude plugin marketplace add Tairitsua/inspect-dependency-source-skill
claude plugin install inspect-dependency-source@inspect-dependency-source
```

## Exact NuGet inspection

```bash
python3 scripts/inspect_dependency_source.py \
  package fetch-nuget Newtonsoft.Json 13.0.3
python3 scripts/inspect_dependency_source.py \
  resolve Newtonsoft.Json --ref 13.0.3 --json
```

The real replay resolves to
[`0a2e291c0d9c0c7675d445703e51750363a549ef`](https://github.com/JamesNK/Newtonsoft.Json/commit/0a2e291c0d9c0c7675d445703e51750363a549ef).
The local JSON result supplies the validated `source_path`, package version,
resolution kind, verification state, and expected and observed commits used for
inspection.

## Safety boundaries

- No telemetry; catalog data and source remain local.
- The dashboard binds to `127.0.0.1`, serves no mutation endpoints, and makes no
  external browser requests.
- Explicit package and ref requests fail closed.
- Managed source is read-only and reusable across projects and agents.
- Repository removal must be previewed with
  `repo remove <query> --plan --json` and paused for explicit authorization.
- Removal execution is bound to the exact `repository.id` and snapshot token from
  the preview, rejects changed plans, serializes against acquisition, and verifies
  that every previously existing preserved path still exists.
- Purge refuses any plan that cannot prove registered local sources are disjoint
  from catalog-managed storage.
- Credentials are sanitized before persistence and redacted from diagnostics,
  JSON, and dashboard APIs.

## Supported sources and requirements

- GitHub repositories, generic Git remotes, existing local source trees, and
  exact NuGet versions.
- Python 3.11 or newer with standard-library `sqlite3`.
- Git for Git-backed registration and fetches.
- Network access only for requested remote or package work.

## Known limits

- npm, PyPI, Maven, and Cargo package metadata are not resolved in v1.0.0; their
  repositories can still be registered through Git or local source.
- A `heuristic_tag` is a lead and does not establish an exact package-to-source
  mapping without independent verification.
- CLI JSON and dashboard APIs contain local paths and operational metadata; keep
  raw payloads out of project files, logs, and release artifacts by default.
