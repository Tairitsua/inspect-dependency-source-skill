# Inspect Dependency Source v1.0.0 — draft release notes

> Draft only. Publishing, tagging, and creating a GitHub release require separate authorization.

## Why this release exists

Coding agents can reason confidently inside a project and still inspect the wrong revision at a dependency boundary. This release turns dependency-source lookup into a conservative evidence workflow: exact request, reusable source, verified commit, and a Source Receipt that states what the evidence can prove.

## Highlights

1. **Exact refs fail closed.** A missing tag or commit never silently becomes the default branch.
2. **One catalog serves every project and agent.** Verified source is downloaded once and reused from a user-level store.
3. **Source Receipts preserve the proof boundary.** `PROVEN`, `CANDIDATE`, and `BLOCKED` are derived from provenance, integrity, and commit agreement.

## First proof

```bash
python3 scripts/inspect_dependency_source.py package fetch-nuget Newtonsoft.Json 13.0.3
python3 scripts/inspect_dependency_source.py \
  resolve Newtonsoft.Json --ref 13.0.3 --receipt
```

The real replay resolves to [`0a2e291c0d9c0c7675d445703e51750363a549ef`](https://github.com/JamesNK/Newtonsoft.Json/commit/0a2e291c0d9c0c7675d445703e51750363a549ef). See the checked-in [Source Receipt](../../examples/newtonsoft-json-13.0.3/source-receipt.md).

## Safety boundaries

- No telemetry; catalog data and source remain local.
- Dashboard binds to `127.0.0.1`, serves no mutation endpoints, and makes no external browser requests.
- Explicit package/ref requests fail closed.
- Managed source is read-only shared evidence.
- Repository removal must be previewed with `repo remove <query> --plan --json` and paused for explicit authorization.
- Removal execution is bound to the exact `repository.id` and snapshot token returned by that preview, rejects intervening deletion-set or visible plan changes, serializes against registration and acquisition, and post-verifies every previously existing preserved path.
- Purge refuses any plan that cannot prove registered local sources are disjoint from catalog-managed storage.

## Supported sources and requirements

- GitHub repositories, generic Git remotes, existing local source trees, and exact NuGet versions.
- Python 3.11 or newer with standard-library `sqlite3`.
- Git for Git-backed registration and fetches.
- Network access only for requested remote or package work.

## Known limits

- npm, PyPI, Maven, and Cargo package metadata are not resolved in v1.0.0; their repositories can still be registered through Git or local source.
- A `heuristic_tag` is a lead, never proof.
- Source Receipts omit local identity details but still reveal repository and package names.

## Verification evidence

- Full unit/integration suite on the minimum and current supported Python versions.
- Focused Source Receipt and removal-safety contracts.
- Real Chromium validation of the bilingual localhost dashboard.
- Candidate Skill installation and Agent Skills structure validation.
- Real network-backed Newtonsoft.Json replay plus a missing-ref failure replay.
